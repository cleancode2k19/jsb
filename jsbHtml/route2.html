In JS where data types can be compared by value are known as primitive, but where it can’t be compared with value called as non primitive. Only Object type is non primitive, where value can accessed by key.
As Javascript is loosely typed, we don’t need to explicitly define variable type. It automatically process by assigned values.

Boolean: stores in Stack

It checks for true or false, mostly for conditions.
var checkFlag = true;
(checkFlag ) ? console.log(‘checkFlag value is true’) : console.log(‘checkFlag value is false’);

NULL: stores in Stack

It is null value assigned to a variable. Difference between empty and Null : empty is a blank string, but typeof null can be object.

Undefined: stores in Stack

stores in Stack The typeof a declared variable without any assigned value known as Undefined. Mostly while using array object values, it’s essential to put a condition whether array is not undefined. Or page load will break by error”trying to access abc key from undefined xyz array”.

Number: stores in Stack

In JS a variable typeof number covers positive integer, negative number and flots/decimals.
EX: var intVal = 5, negativeVal=-2, floatVal=2.3;
 alert(typeof intVal);
 alert(typeof negativeVal);
 alert(typeof floatVal);

String: stores in Stack

In JS a variable typeof String can hold both characters as well as numbers. The values are assigned with in quotes.
Difference between string and number
EX: var a=4, b=5; alert(a+b); //9
 var c=’4’+’5’; alert(c+d);//45
But exception for ‘-’, because of auto type casting it does the subtraction, irrespective of type(string or number)

Symbol: stores in Stack

It is ES6 type, basically supported by advance browsers. It is not process in older browsers.A symbol is a unique and immutable data type and may be used as an identifier for object properties. The symbol object is an implicit object wrapper for the symbol primitive data type.
EX: let name = Symbol("name");
 alert(name.toString());

EX: var myObject = {“key”:”value”};
